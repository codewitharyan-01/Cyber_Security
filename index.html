Q1
Define System and Web Vulnerability. Explain different types of vulnerabilities.

System Vulnerability: A weakness or flaw in a computer system's design, implementation, or operation that could be exploited to violate the system's security policy.

Web Vulnerability: A security weakness in web applications, web servers, or web browsers that can be exploited by attackers to gain unauthorized access or perform malicious actions.

Types of Vulnerabilities:

  • Input Validation Flaws: SQL injection, XSS, command injection
  • Authentication Weaknesses: Weak passwords, session management issues
  • Authorization Failures: Privilege escalation, access control issues
  • Configuration Errors: Default settings, unnecessary services
  • Buffer Overflows: Memory corruption vulnerabilities
  • Information Disclosure: Exposing sensitive data or system information
Q2
Explain Nmap and its different functionalities with commands.

Nmap (Network Mapper) is a free and open-source network scanning tool used for network discovery, security auditing, and port scanning.

Key Functionalities:

  • Host Discovery: Finding live hosts on a network
  • Port Scanning: Identifying open ports and services
  • Service Version Detection: Determining service versions
  • OS Detection: Identifying operating systems
  • Scriptable Interaction: Using NSE (Nmap Scripting Engine)

Common Nmap Commands:

  • nmap -sP 192.168.1.0/24 - Ping scan to find live hosts
  • nmap -sS 192.168.1.1 - SYN scan (stealth scan)
  • nmap -sT 192.168.1.1 - TCP connect scan
  • nmap -sU 192.168.1.1 - UDP port scan
  • nmap -sV 192.168.1.1 - Service version detection
  • nmap -O 192.168.1.1 - OS detection
  • nmap -A 192.168.1.1 - Aggressive scan (all features)
  • nmap -p 1-65535 192.168.1.1 - Scan all ports

Q4
Explain Metasploit Framework and its components.

Metasploit Framework is an open-source penetration testing framework that provides tools for developing and executing exploit code against remote target machines.

Key Components:

  • msfconsole: Main command-line interface for Metasploit
  • Exploits: Code that takes advantage of vulnerabilities
  • Payloads: Code that runs on target system after exploitation
  • Auxiliary: Modules for scanning, fuzzing, and other non-exploitation tasks
  • Encoders: Encode payloads to avoid detection
  • Nops: No-operation generators for payload alignment
  • Post-exploitation: Modules for maintaining access and gathering information

Basic Metasploit Workflow:

  1. Launch Metasploit: msfconsole
  2. Search for exploit: search exploit_name
  3. Use exploit: use exploit/path
  4. Set target options: set RHOST target_ip
  5. Set payload: set PAYLOAD payload_name
  6. Execute: exploit or run
Q5
What is SQL Injection? Explain with example and prevention methods.

SQL Injection is a code injection technique that exploits security vulnerabilities in a web application's database layer by manipulating SQL queries.

How SQL Injection Works:

  • Attacker inputs malicious SQL code through user input fields
  • The application constructs SQL queries using this input
  • Malicious SQL gets executed on the database
  • Attacker can read, modify, or delete database data

Example of SQL Injection:

Vulnerable Code:

SELECT * FROM users WHERE username = '$username' AND password = '$password'

Malicious Input:

  • Username: admin'--
  • Password: anything

Resulting Query:

SELECT * FROM users WHERE username = 'admin'--' AND password = 'anything'

The -- comments out the password check, allowing unauthorized access.

Prevention Methods:

  • Parameterized Queries: Use prepared statements with bound parameters
  • Input Validation: Validate and sanitize all user inputs
  • Stored Procedures: Use database stored procedures
  • Least Privilege: Give database users minimal required permissions
  • Web Application Firewall (WAF): Filter malicious requests
  • Regular Security Testing: Conduct penetration testing and code reviews
Q6
Define Digital Forensics and explain its phases.

Digital Forensics is the process of identifying, preserving, analyzing, and presenting digital evidence in a legally admissible manner for use in legal proceedings.

Key Objectives:

  • Preserve evidence in its original state
  • Reconstruct digital events and timelines
  • Identify responsible parties
  • Support legal proceedings with technical evidence

Phases of Digital Forensics:

  1. Identification: Recognizing potential sources of digital evidence
  2. Preservation: Isolating and securing digital evidence to prevent alteration
  3. Collection: Acquiring digital evidence using proper forensic procedures
  4. Examination: Processing collected data to identify relevant information
  5. Analysis: Interpreting examined data to draw conclusions
  6. Presentation: Reporting findings in clear, understandable format
  7. Decision: Using forensic results for legal or administrative action

Common Digital Forensics Tools:

  • Autopsy: Digital forensics platform for hard drive investigation
  • Sleuth Kit: Collection of command-line forensics tools
  • FTK Imager: Tool for creating forensic images
  • Wireshark: Network protocol analyzer
  • Volatility: Memory forensics framework
Q7
Explain IT Act, 2000 and its important sections.

Information Technology Act, 2000 (IT Act, 2000) is the primary law in India dealing with cybercrime and electronic commerce. It provides legal recognition to electronic transactions and digital signatures.

Key Objectives:

  • Legal recognition of electronic transactions
  • Facilitation of electronic filing with government agencies
  • Amendment of Indian Penal Code for digital crimes
  • Protection of online privacy and data
  • Establishment of cyber appellate tribunal

Important Sections of IT Act, 2000:

Section 43 - Penalty for Damage to Computer System:

  • Compensation up to 1 crore for unauthorized access
  • Covers data theft, introducing viruses, damaging systems

Section 65 - Tampering with Computer Source Documents:

  • Punishment: Up to 3 years imprisonment or 2 lakh fine
  • Concealing, destroying, or altering source code

Section 66 - Computer Related Offenses:

  • Punishment: Up to 3 years imprisonment or 5 lakh fine
  • Dishonestly or fraudulently doing any act referred to in section 43

Section 66C - Identity Theft:

  • Punishment: Up to 3 years imprisonment and/or fine up to 1 lakh
  • Fraudulently using electronic signature, password, or unique identification

Section 66D - Cheating by Personation:

  • Punishment: Up to 3 years imprisonment and/or fine up to 1 lakh
  • Cheating by pretending to be someone else using computer resources

Section 66E - Privacy Violation:

  • Punishment: Up to 3 years imprisonment and/or fine up to 2 lakh
  • Capturing, publishing, or transmitting private images without consent

Section 67 - Publishing Obscene Information:

  • Punishment: Up to 3 years imprisonment and/or fine up to 5 lakh
  • Publishing or transmitting obscene material in electronic form
Q8
Explain Buffer Overflow attack with prevention techniques.

Buffer Overflow is a software vulnerability that occurs when a program tries to store more data in a buffer than it can hold, causing the excess data to overflow into adjacent memory locations.

How Buffer Overflow Works:

  1. Program allocates fixed-size buffer in memory
  2. Attacker provides input larger than buffer capacity
  3. Excess data overwrites adjacent memory
  4. Can overwrite return address, function pointers, or critical data
  5. Attacker can execute arbitrary code or crash the system

Types of Buffer Overflow:

  • Stack Overflow: Overwriting stack memory
  • Heap Overflow: Overwriting heap memory
  • Format String Attack: Exploiting format string functions
  • Integer Overflow: Arithmetic operations causing overflow

Prevention Techniques:

  • Input Validation: Check input length before copying to buffers
  • Safe Functions: Use strncpy() instead of strcpy()
  • Stack Canaries: Place special values between buffer and return address
  • ASLR (Address Space Layout Randomization): Randomize memory layout
  • Data Execution Prevention (DEP): Mark memory regions as non-executable
  • Compiler Protections: Use stack protection mechanisms
  • Regular Code Reviews: Identify and fix vulnerable code
